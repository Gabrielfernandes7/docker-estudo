# Construindo imagens de container

Imagens de container são pacotes de executáveis que incluem tudo o que é necessário para executar uma aplicação: código, runtime, ferramentas do sistema, bibliotecas e configurações.

Dockerfile → build → Image → run → Container

Dockerfile
É a receita. Define tudo que a aplicação precisa.

Image
É o resultado da receita. Um pacote pronto.

Container
É a aplicação rodando.

Exemplos de imagem docker criadas no repositório: https://github.com/dockersamples
